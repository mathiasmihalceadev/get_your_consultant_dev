# Custom Icon Package Guide

## Property Report Analysis Platform

This document specifies the requirements for a custom icon package designed for a modern property analysis and reporting platform. The icons should feel premium, professional yet approachable, and consistently integrated with the platform's visual language.

---

## Brand Identity & Visual Style

### Color Palette

- **Primary**: `#303048` (Deep Navy/Purple) — Authority, trust
- **Secondary**: `#f5915d` (Warm Orange) — Accessibility, warmth, action
- **Tertiary**: `#0073f0` (Vibrant Blue) — Energy, clarity, insights
- **Neutral**: `#64748b` (Slate Gray) — Hierarchy, balance
- **Accents**: Emerald green (`#10b981`) for success/positive states

### Typography

- Font: **Plus Jakarta Sans** (modern, geometric, friendly)
- Weights: 400 (regular), 600 (semibold), 700 (bold)

### Design System Characteristics

1. **Modern & Minimalist**: Clean lines, subtle complexity, not overly flat
2. **Warm Professionalism**: Friendly but authoritative, approachable yet trustworthy
3. **Geometric Foundation**: Use of circles, squares, and hexagons as structural elements
4. **Duotone/Layered**: Icons should support duotone rendering (2-3 color layers for visual depth)
5. **Consistent Stroke Weight**: 1.5-2px strokes for sizes 16-24px
6. **Rounded Corners**: Use 2-3px border radius on geometric shapes for softness
7. **Breathing Space**: Generous padding within icon boundaries (20-30% of viewBox)

### Visual Tone

- **Not generic**: Avoid direct Phosphor Icon clones
- **Slightly playful**: Character-driven, not sterile
- **Scalable**: Work seamlessly at 16px, 20px, 24px, and 32px
- **Accessibility-first**: High contrast, clear forms at all sizes

## Technical Specifications

### File Formats & Deliverables

1. **SVG Format** (primary)
    - Viewbox: 24x24 or 32x32 (scale-agnostic)
    - Stroke-based (not filled) for customization flexibility
    - No text rendering
    - Optimized, clean SVG code

2. **Optional Variations**
    - Duotone versions with layer separation (path classes: `.icon-primary`, `.icon-secondary`)
    - Outline (stroke) and filled versions for select icons
    - Animated versions (spinners, loaders) with CSS animation hints

### Color Application

#### Monochrome Usage

```css
/* Default: Brand Primary */
.icon {
    color: #303048;
}

/* On secondary backgrounds */
.icon.secondary {
    color: #f5915d;
}

/* On tertiary backgrounds */
.icon.tertiary {
    color: #0073f0;
}

/* Disabled state */
.icon.disabled {
    color: #64748b;
    opacity: 0.5;
}
```

#### Duotone Usage

```css
/* For dual-color icons */
.icon.duotone {
    --icon-primary: #303048;
    --icon-secondary: #f5915d;
}

/* Dynamic status coloring */
.icon.success.duotone {
    --icon-secondary: #10b981;
}

.icon.error.duotone {
    --icon-secondary: #ef4444;
}
```

### Sizing Guidelines

- **16px**: Administrative, compact UI, small badges
- **20px**: Form inputs, small buttons, navigation items
- **24px**: Standard, default UI components, buttons
- **32px**: Hero sections, large buttons, prominent calls-to-action
- **48px+**: Marketing, landing page hero elements

### Stroke Weight

- **16px icons**: 1.5px stroke
- **20-24px icons**: 1.75px stroke
- **32px+ icons**: 2px stroke
- **Consistency**: Maintain optical weight across sizes

### Padding/Breathing Space

- Minimum: 2px gutter from viewBox edge
- Recommended: 3-4px gutter for visual balance
- Icons should feel centered with natural whitespace

## Design Principles & Constraints

### DO ✅

- Draw inspiration from nature, architecture, and geometric forms
- Use consistent stroke weight and minimal line intersections
- Employ subtle details that reveal personality on close inspection
- Design for clarity first; aesthetics second
- Consider animation potential (rotations, bounces, morphs)
- Test legibility at 16px (smallest size)
- Use modular components (circles, squares) as structural elements
- Balance complexity: detail without clutter

### DON'T ❌

- Don't copy Phosphor, Feather, or other existing icon libraries directly
- Don't use complex gradients or multiple colors within single icon
- Don't include text, labels, or numbers inside icons
- Don't over-detail; aim for ~3-5 semantic elements per icon
- Don't use thin strokes (<1.5px); they'll disappear at small sizes
- Don't make icons ambiguous or require explanation
- Don't include sharp points or overly complex curves
- Don't ignore dark/light background contrast
